**Category include constant (reuse across all methods that return transactions):**
```typescript
const categoryInclude = { category: { select: { id: true, name: true, type: true, icon: true, color: true } } }
```

**Methods:**

1. `static async getAll(userId: string, query: GetAllQuery)` — Paginated list with filters.
   - Destructure: `{ page = 1, limit = 20, type, categoryId, dateFrom, dateTo, search } = query`
   - Compute `skip = (page - 1) * limit`
   - Build `where` object: always `userId`, conditionally add `type`, `categoryId`, `transactionDate` range (`gte`/`lte` using `new Date()`), and `search` as `OR: [{ description: { contains: search, mode: "insensitive" as const } }, { notes: { contains: search, mode: "insensitive" as const } }]`
   - Use `prisma.$transaction([prisma.transaction.findMany({ where, include: categoryInclude, orderBy: { transactionDate: "desc" }, skip, take: limit }), prisma.transaction.count({ where })])` for atomic pagination
   - Return `{ data, total, page, limit, totalPages: Math.ceil(total / limit) }`

2. `static async getById(userId: string, id: string)` — Single transaction.
   - `prisma.transaction.findUnique({ where: { id, userId }, include: categoryInclude })`
   - If null, throw `new NotFoundError("Transaction doesn't exist")`
   - Return the transaction

3. `static async create(userId: string, input: TransactionCreateInput)` — Create transaction.
   - Destructure `{ categoryId, ...rest } = input`
   - `prisma.transaction.create({ data: { ...rest, userId, ...(categoryId !== undefined ? { categoryId } : {}) }, include: categoryInclude })`
   - Return `status(201, result)`

4. `static async update(userId: string, id: string, input: TransactionUpdateInput)` — Update with patch semantics.
   - Check existence: `prisma.transaction.findUnique({ where: { id, userId } })` — throw `NotFoundError` if null
   - Destructure `{ categoryId, ...rest } = input`
   - `prisma.transaction.update({ where: { id, userId }, data: { ...rest, ...(categoryId !== undefined ? { categoryId } : {}) }, include: categoryInclude })`
   - Return the updated transaction

5. `static async delete(userId: string, id: string)` — Delete single.
   - Check existence: `prisma.transaction.findUnique({ where: { id, userId } })` — throw `NotFoundError` if null
   - `prisma.transaction.delete({ where: { id, userId } })`
   - Return deleted transaction

6. `static async bulkDelete(userId: string, ids: string[])` — Bulk delete.
   - `prisma.transaction.deleteMany({ where: { id: { in: ids }, userId } })`
   - Return the `{ count }` result directly

**CRITICAL anti-patterns to avoid:**
- Do NOT call `.toNumber()` on amount — Prisma Decimal serializes to string automatically
- Do NOT use `TransactionRelationsInputCreate` — it uses Prisma relation connect syntax
- Do NOT omit `userId` from `deleteMany` where clause
- Do NOT omit `include: categoryInclude` from create/update/getById/getAll responses
- For `categoryId` patch semantics: check `categoryId !== undefined` (not truthiness), because `null` is a valid value meaning "clear category"
