## Color

All values reference existing CSS custom properties from `globals.css`. Do not use hardcoded hex.

| Role | CSS Variable | Usage |
|------|-------------|-------|
| Dominant (60%) | `--background` (oklch 98.28% warm white) | Page background, sidebar |
| Secondary (30%) | `--card` (oklch 100% white) | Cards, chart containers, popover surfaces |
| Accent (10%) | `--primary` (oklch 0.527 green) | See reserved list below |
| Destructive | `--destructive` (oklch 0.577 red-orange) | Destructive actions only |

Accent (`--primary` green) reserved for:
- Income value display (positive net balance)
- Income series in bar/area charts (`chart-1` through `chart-5` for category breakdown)
- Active toggle state in time-range selector
- TrendingUp badge when net balance is positive

Expense / negative balance uses `--destructive` (red-orange), not a separate accent.

Chart color series:
- Category breakdown donut: use `--chart-1` through `--chart-5` cycling (already defined in globals.css)
- Monthly trends bar: income series = `--primary`, expense series = `--destructive`

Dark mode: CSS variables switch automatically via `.dark` variant ŌĆö no additional mapping needed.

## Interaction Contract

### SectionCards ŌĆö Balance Summary
- Data source: `GET /dashboard/summary?dateFrom=&dateTo=` (defaults to current month)
- 4 cards: Total Income, Total Expenses, Net Balance, [Savings Rate or placeholder]
- Net Balance positive: primary green, TrendingUpIcon
- Net Balance negative: destructive red, TrendingDownIcon
- Loading state: skeleton card (h-32 per card)
- Empty state: show $0.00 with neutral badge (no trend icon)

### MonthlyTrendsChart ŌĆö Bar Chart
- Data source: `GET /dashboard/trends?months=6`
- Time range toggle: 3m / 6m / 12m (ToggleGroup, default 6m) ŌĆö changes `months` param
- Series: income (primary green) vs expenses (destructive red)
- X-axis: abbreviated month labels (Jan, Feb, ...)
- Tooltip: show both income + expense values for hovered month, formatted with user currency
- Loading state: skeleton (h-[300px])
- Empty state: chart renders with all-zero bars + muted message "No transactions this period"

### CategoryBreakdownChart ŌĆö Donut Chart
- Data source: `GET /dashboard/category-breakdown` (defaults to current month)
- Uses category `color` field from API response as chart fill where possible; fallback to `--chart-N` cycle
- Center label: total expense amount for the period
- Legend: category name + amount, max 6 items visible, overflow truncated with "+N more"
- Loading state: skeleton circle (h-[200px] w-[200px] mx-auto)
- Empty state: empty donut outline + "No spending this month"

### RecentTransactionsWidget ŌĆö List
- Data source: `GET /dashboard/recent?limit=10`
- Columns: category icon+name, description, date, amount (income green / expense red)
- No pagination ŌĆö widget shows exactly limit=10 rows
- Row click: no action (read-only widget; link to /dashboard/transactions is out of scope for this phase)
- Loading state: 5 skeleton rows (h-10 each)
- Empty state: "No recent transactions" with a prompt to add one (link to transactions page)

## Loading and Error States

| State | Pattern |
|-------|---------|
| Loading (initial fetch) | Suspense boundary with `<Skeleton>` per widget, matching widget dimensions |
| Error (API failure) | Inline error message inside card: "Could not load data. Try refreshing." ŌĆö no toast |
| Stale data | TanStack Query default stale behavior (global staleTime from QueryClient config) |
| Zero data | Widget renders with $0 / empty list + contextual empty state copy (see Copywriting) |
